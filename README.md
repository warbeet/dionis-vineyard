# 🍇 Dionis vineyard — PWA для управления виноградником

**Dionis vineyard** — профессиональное прогрессивное веб-приложение (PWA) для коммерческого виноградника. Работает оффлайн, синхронизируется через Firebase, анализирует фото через AI, выдаёт PDF-отчёты.

## ✨ Возможности

| Функция | Описание |
|---|---|
| 📊 **Дашборд** | Сводная статистика, риски, лента событий |
| 🗺 **Участки** | Карточки блоков с сортами, подвоем, формировкой |
| 📸 **Фотоотчёты** | Загрузка с камеры/галереи, автосжатие, AI-анализ |
| 🤖 **AI-анализ через OpenRouter** | GPT-4o, Claude 3.5, Gemini, Llama, Qwen — на выбор |
| 🌦 **Онлайн-погода Open-Meteo** | Прогноз 7 дней, алерты заморозков/жары/ливней. Бесплатно, без API-ключа |
| 📔 **Журнал работ** | Все операции с часами и стоимостью |
| 💊 **Болезни и обработки** | Учёт препаратов со сроками ожидания |
| 📋 **План работ** | Автогенерация по фенофазе |
| 🍷 **Урожай и затраты** | Brix, pH, кислотность, расходы по категориям |
| 📄 **PDF-отчёты** | Еженедельные, по участку, сезонные, по обработкам |
| 👥 **Команда** | Роли: владелец, агроном, рабочий, наблюдатель |
| 📱 **PWA** | Установка на телефон, оффлайн-режим, push (готово к подключению) |

## 🚀 Быстрый старт (3 минуты)

### Вариант A: локальный режим (без регистрации)

1. Откройте `index.html` в браузере (или загрузите на любой хостинг — GitHub Pages, Netlify)
2. На экране входа нажмите **«Использовать без аккаунта»**
3. Готово! Данные хранятся в браузере (localStorage)

### Вариант B: с Firebase (рекомендуется)

1. **Создайте проект Firebase** (5 минут):
   - Зайдите на https://console.firebase.google.com
   - Нажмите «Add project» → введите имя
   - В созданном проекте: **Build → Authentication** → Get Started → включите **Email/Password** и **Google**
   - **Build → Firestore Database** → Create → выберите регион → начните в **Production mode**
   - Перейдите в **Rules** Firestore и вставьте:
   ```
   rules_version = '2';
   service cloud.firestore {
     match /databases/{database}/documents {
       match /vineyards/{id} {
         allow read, write: if request.auth != null && (
           resource == null ||
           request.auth.uid == resource.data.ownerId ||
           request.auth.token.email in resource.data.members[*].email
         );
       }
       match /users/{uid} {
         allow read, write: if request.auth != null && request.auth.uid == uid;
       }
       match /invitations/{id} {
         allow read, write: if request.auth != null;
       }
     }
   }
   ```
   - **Project settings (⚙️) → General** → Your apps → нажмите `</>` (Web)
   - Зарегистрируйте приложение, скопируйте объект `firebaseConfig`

2. **Настройте приложение**:
   - Откройте `index.html`
   - Зайдите в локальный режим
   - Перейдите в **Настройки → Firebase**, вставьте JSON-конфиг (только содержимое объекта `{...}`)
   - Нажмите «Сохранить» — приложение перезагрузится
   - Зарегистрируйтесь / войдите через Google

### Настройка AI (опционально)

1. Получите API-ключ на https://openrouter.ai/keys (есть бесплатные модели)
2. Пополните баланс ($5 хватит надолго) или используйте бесплатные модели
3. **Настройки → OpenRouter API** → вставьте ключ → выберите модель → Сохранить
4. Теперь в фотоотчётах появится кнопка **«🤖 Сохранить + AI-анализ»**

**Рекомендуемые модели для агро-анализа:**
- `openai/gpt-4o` — лучший баланс качества и цены ($2.50/$10 за 1M токенов)
- `google/gemini-flash-1.5` — самый дешёвый ($0.075/$0.30)
- `anthropic/claude-3.5-sonnet` — самый детальный анализ

## 👥 Многопользовательский режим

1. Войдите как владелец → раздел **Команда**
2. Скопируйте код виноградника (формат `VYD-XXXX-XXXX`)
3. Создайте приглашение для email участника, выберите роль
4. Передайте код участнику
5. Участник регистрируется в приложении → раздел **Команда** → «Подключиться к чужому виноградника» → вводит код

**Роли:**
- **👑 Владелец** — полный доступ
- **🧑‍🌾 Агроном** — просмотр + рекомендации
- **👷 Рабочий** — задачи и фотоотчёты
- **👁 Наблюдатель** — только чтение

## 💾 Безопасность данных

- Локально: всё в `localStorage` браузера
- Firebase: данные в Firestore, доступ только аутентифицированным пользователям виноградника
- **Резервная копия**: Настройки → «Экспорт всех данных» (JSON) — храните копии!
- API-ключи OpenRouter хранятся **только локально** в браузере, не передаются никуда кроме openrouter.ai

## ⚠️ Лимиты и нюансы

- **Лимит Firestore**: 1 МБ на документ. Фото сжимаются до ~100-200 КБ. При >10 МБ данных лучше выносить фото в Firebase Storage (можно дописать)
- **OpenRouter**: тарификация по токенам, 1 фото ≈ 1000-2000 токенов
- **Open-Meteo**: бесплатно до 10 000 запросов/день — более чем достаточно
